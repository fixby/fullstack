# QA 问题分类

## 严重性级别

| 严重性 | 定义 | 示例 |
|--------|------|------|
| **critical** | 阻塞核心工作流、导致数据丢失或导致应用崩溃 | 表单提交导致错误页面、结账流程中断、数据未经确认被删除 |
| **high** | 主要功能损坏或不可用,无变通方法 | 搜索返回错误结果、文件上传静默失败、身份验证重定向循环 |
| **medium** | 功能正常但有明显问题,存在变通方法 | 页面加载缓慢(>5s)、缺少表单验证但提交仍然有效、布局仅在移动设备上损坏 |
| **low** | 轻微的装饰或润色问题 | 页脚中的拼写错误、1px 对齐问题、悬停状态不一致 |

## 类别

### 1. 视觉/UI
- 布局损坏(元素重叠、文本被裁剪、水平滚动条)
- 图像损坏或缺失
- z-index 不正确(元素出现在其他元素后面)
- 字体/颜色不一致
- 动画故障(卡顿、过渡不完整)
- 对齐问题(偏离网格、间距不均匀)
- 深色模式 / 主题问题

### 2. 功能
- 链接损坏(404、错误目标)
- 按钮失效(点击无反应)
- 表单验证(缺失、错误、被绕过)
- 重定向不正确
- 状态未持久化(刷新时数据丢失、后退按钮)
- 竞态条件(双重提交、陈旧数据)
- 搜索返回错误或无结果

### 3. UX
- 导航混乱(无面包屑、死胡同)
- 缺少加载指示器(用户不知道正在发生什么)
- 交互缓慢(>500ms 无反馈)
- 错误消息不明确("Something went wrong" 无详细信息)
- 破坏性操作前无确认
- 页面间交互模式不一致
- 死胡同(无法返回、无下一步操作)

### 4. 内容
- 拼写错误和语法错误
- 过时或不正确的文本
- 占位符 / lorem ipsum 文本遗留
- 文本截断(被切断而无省略号或"更多")
- 按钮或表单字段上的标签错误
- 缺少或无帮助的空状态

### 5. 性能
- 页面加载缓慢(>3 秒)
- 滚动卡顿(丢帧)
- 布局偏移(加载后内容跳动)
- 过多的网络请求(单个页面上 >50 个)
- 大型未优化图像
- 阻塞 JavaScript(加载期间页面无响应)

### 6. 控制台/错误
- JavaScript 异常(未捕获的错误)
- 网络请求失败(4xx、5xx)
- 弃用警告(即将中断)
- CORS 错误
- 混合内容警告(HTTPS 上的 HTTP 资源)
- CSP 违规

### 7. 可访问性
- 图像上缺少 alt 文本
- 未标记的表单输入
- 键盘导航损坏(无法 Tab 到元素)
- 焦点陷阱(无法退出模态框或下拉菜单)
- 缺少或不正确的 ARIA 属性
- 颜色对比度不足
- 屏幕阅读器无法访问内容

## 每页探索清单

对于 QA 会话期间访问的每个页面:

1. **视觉扫描** — 拍摄带注释的截图(`snapshot -i -a -o`)。查找布局问题、图像损坏、对齐。
2. **交互元素** — 点击每个按钮、链接和控件。每个是否都按其说明执行?
3. **表单** — 填写并提交。测试空提交、无效数据、边缘情况(长文本、特殊字符)。
4. **导航** — 检查所有进出路径。面包屑、后退按钮、深层链接、移动菜单。
5. **状态** — 检查空状态、加载状态、错误状态、完整/溢出状态。
6. **控制台** — 交互后运行 `console --errors`。是否有新的 JS 错误或失败的请求?
7. **响应性** — 如果相关,检查移动和平板视口。
8. **身份验证边界** — 注销时会发生什么?不同的用户角色?
